# 🚀 The Silicon Valley Workshop Series
### Hosted by AWS Cloud Club – FAST NUCES Peshawar

Welcome to the official repository for **The Silicon Valley Workshop**, a 5-week intensive series designed to bridge the gap between university logic and industry-ready production. As an organization, we prioritize **"Skills over Events,"** moving students from basic coding to deploying live, world-accessible applications.

---

## 📌 Workshop Overview
This workshop series was designed to make students familier with cloud and the modern MLOps pipeline.

- **Goal**: Build, containerize, and deploy a production-ready API.
- **Philosophy**: Interactive, hands-on learning with a focus on professional standards.

---

## 📂 Repository Structure

Each folder contains the presentation slides, code snippets, and lab tasks for that specific session:

| Folder | Session Title | Key Focus Area |

---

## 🛠 Tech Stack
We utilize modern, industry-standard tools to ensure students are ready for top-tier internships:

- **Environment**: WSL2 (Ubuntu), Linux Terminal 
- **Package Management**: `uv` (Rust-based ultra-fast manager) 
- **Backend**: Python 3.11+, FastAPI, Pydantic 
- **Cloud**: Amazon EC2, IAM, Security Groups 
- **DevOps**: Docker, Git, GitHub Actions 
---

## 🚀 Session Highlights

### Session 1 & 2: Building the Engine
We transitioned from GUI-based coding to a professional CLI workflow. Using **`uv`**, we established isolated virtual environments to build **RESTful APIs** that follow industry principles like statelessness and client-server decoupling.

### Session 3 & 4: Security & Portability
Students moved their code into **Docker** "shipping containers" to solve the "works on my machine" problem. We configured secure **AWS IAM** policies and automated workflows with **GitHub Actions**.

### Session 5: The Grand Finale
The series concluded with a live **Production Launch**. We took raw ML logic, wrapped it in a FastAPI endpoint, and deployed it on an **AWS EC2** instance, making the applications accessible via public endpoints.
